# Tài liệu chuẩn

Mục tiêu của bộ tài liệu này là giữ cho việc debug, thêm tính năng, sửa lỗi và chỉnh giao diện đi theo cùng một chuẩn.

Đọc theo thứ tự:

1. `docs/architecture.md`
2. `docs/rules.md`
3. `docs/feature-workflow.md`
4. `docs/debugging-playbook.md`
5. `docs/ui-guidelines.md`
6. `docs/data-contracts.md`

## Khi nào dùng tài liệu nào

- Muốn hiểu repo: `docs/architecture.md`
- Muốn biết cách làm việc an toàn: `docs/rules.md`
- Muốn thêm hoặc sửa tính năng: `docs/feature-workflow.md`
- Muốn dò lỗi: `docs/debugging-playbook.md`
- Muốn chỉnh giao diện: `docs/ui-guidelines.md`
- Muốn đổi format dữ liệu: `docs/data-contracts.md`
